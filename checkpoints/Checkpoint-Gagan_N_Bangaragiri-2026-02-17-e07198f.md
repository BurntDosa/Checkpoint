# Checkpoint Architecture Update: CI/CD-Centric Checkpoint Generation

## Context
The system previously relied on **local git hooks** and bulk context updates (`--onboard`) to generate checkpoints. This approach had limitations:
- Inconsistencies between local and remote environments.
- Redundant processing if hooks and CI/CD both generated checkpoints.
- Lack of granularity in context updates.

## Changes
### 1. GitHub Workflow (`checkpoint.yml`)
- **Before**: Single `--onboard` command for bulk updates.
- **After**:
  - Dynamically fetches pushed commits using `git rev-list`.
  - Processes each commit individually via `python main.py --commit <hash>`.
  - Retains `--catchup-all` for developer summaries but separates it from commit processing.

### 2. Setup Configuration (`setup.py`)
- **Disabled by Default**:
  - `install_git_hook = False` (previously `True`).
  - `auto_catchup = False` (previously `True`).
- **User Messaging**:
  - Clarifies that checkpoints are now generated by GitHub Actions on push.
  - Emphasizes "Core features" (Vector DB, Diagrams) without local hooks.

### 3. New `main.py` Functionality (Inferred)
- Added `--commit <hash>` argument to generate checkpoints for specific commits.

## Impact
### ✅ Benefits
- **Consistency**: Checkpoints generated in a controlled CI/CD environment.
- **Granularity**: Per-commit processing reduces noise and improves tracking.
- **Simplified Local Setup**: No need for git hooks or auto-catchup locally.

### ⚠️ Trade-offs
- **Latency**: Checkpoints are only generated on push (not locally).
- **CI/CD Dependency**: Requires GitHub Actions to be operational.

### 🔧 Developer Workflow Changes
- **Local Changes**: No automatic checkpoints; must push to trigger updates.
- **Catchup Summaries**: Generated after commit processing, ensuring up-to-date information.

## Technical Details
- **Commit Detection Logic**:
  ```bash
  # New branch: all commits
  git rev-list ${{ github.sha }}
  # Existing branch: commits between before/after SHA
  git rev-list ${{ github.event.before }}..${{ github.sha }}
  ```
- **Error Handling**: Silences `git rev-list` errors (e.g., no new commits).
- **CLI Changes**: `--commit <hash>` replaces `--onboard` for granular updates.
